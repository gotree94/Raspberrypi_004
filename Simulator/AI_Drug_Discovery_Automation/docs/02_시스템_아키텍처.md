# 2. 시스템 아키텍처

## 2.1 전체 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                    사용자 인터페이스                      │
│  (CLI / REST API / WebSocket / Jupyter Notebook)        │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│                    Pipeline Layer                        │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐   │
│  │Orchestr. │ │Workflow  │ │State     │ │Job       │   │
│  │          │ │Manager   │ │Manager   │ │Scheduler │   │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘   │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│                    Service Layer                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐   │
│  │AlphaFold │ │Molecular │ │Screening │ │  ADMET   │   │
│  │Wrapper   │ │  (RDKit) │ │Similarity│ │Predictor │   │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘   │
│  ┌──────────┐ ┌──────────┐ ┌───────────────────────┐   │
│  │Molecular │ │  Docking │ │  Visualization        │   │
│  │Generation│ │AutoDock  │ │  (charts, viewer)     │   │
│  └──────────┘ └──────────┘ └───────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│                    Data Layer                            │
│  compounds/  proteins/  results/  models/  configs/     │
└─────────────────────────────────────────────────────────┘
```

## 2.2 모듈 의존성

```
alphafold ──┐
            ├──▶ pipeline ──▶ api
molecular ──┤                │
screening ──┤                │
admet ──────┤                │
generation ─┤                │
docking ────┘                │
                    visualization
```

## 2.3 설정 관리

- **ConfigManager**: YAML/JSON 설정 파일 + 환경변수(`AI_DRUG_*` prefix) 오버라이드
- **AppConfig**: 모든 하위 설정을 포함하는 루트 dataclass
- **get_config()**: 전역 싱글톤 접근자

## 2.4 데이터 흐름

1. 입력: SMILES 문자열 / FASTA 서열 / PDB 파일
2. 처리: 각 모듈이 순차적/병렬로 데이터 처리
3. 저장: 결과를 data/results/에 JSON/PDBQT/이미지로 저장
4. 출력: REST API 응답 / 파일 다운로드 / 시각화
