# v3 — 장비 제원 반영 시뮬레이터 고도화

> 실제 대응 장비의 모터·센서 구성을 추정하여 각 시뮬레이터 GUI에 반영.
> v3 목표: 단순 상태 표시를 넘어 **"무슨 모터가 어떻게 움직이고, 어떤 센서가 무슨 값을 읽는지"** 를 실시간으로 표시.

---

## 개요

v1/v2의 시뮬레이터는 단순 진행 바와 텍스트 로그만 표시했다면,
v3는 각 장비의 **실물 제원에 기반한 모터·센서 구성을 GUI에 표시**하고
명령 실행 시 이들의 상태가 변화하는 모습을 보여준다.

---

## 7. 테스트 자동 실행

`test_runner.py`는 장비 시뮬레이터를 **자동으로 실행**하고 테스트를 진행합니다.

```bash
cd v3

# 장비 자동 실행 + 전체 통합 테스트 (T1~T6)
python test_runner.py --auto

# 장비 자동 실행 + 특정 테스트만
python test_runner.py T3 --auto

# 이미 장비가 실행 중인 경우 (자동 실행 없이)
python test_runner.py T3

# 개별 장비 단위 테스트
python test_runner.py --unit --auto
```

**동작 흐름:**
1. `launch_devices()` — 4개 장비 시뮬레이터를 `subprocess`로 백그라운드 실행
2. `wait_for_devices()` — 각 장비 TCP 포트(50051~50054) readiness 폴링 (최대 30s)
3. `run_closed_loop_plan()` — 각 테스트 JSON에 따라 Closed-Loop 실행
4. `terminate_devices()` — 종료 시 `atexit` 등록되어 자동 cleanup
