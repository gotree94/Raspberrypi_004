# 자율 주행 lane detection 프로젝트

## 개요

카메라로 도로 영상을 촬영하고, NVIDIA End-to-End Learning 모델을 사용하여 스티어링 각도를 예측하는 자율 주행 프로젝트입니다.

**목표**: 이미지를 입력으로 받아 스티어링 각도(90~179도)를 예측

## 모델 아키텍처 (NVIDIA End-to-End)

### CNN 특징 추출기

| 레이어 | 타입 | 입력 | 출력 | 커널 | 스트라이드 | 활성화 |
|--------|------|------|------|------|-----------|--------|
| 1 | Conv2d | 3 | 24 | 5x5 | 2 | ELU |
| 2 | Conv2d | 24 | 36 | 5x5 | 2 | ELU |
| 3 | Conv2d | 36 | 48 | 5x5 | 2 | ELU |
| 4 | Conv2d | 48 | 64 | 3x3 | 1 | ELU |
| - | Dropout | - | - | p=0.2 | - | - |
| 5 | Conv2d | 64 | 64 | 3x3 | 1 | ELU |

### MLP 헤드

| 레이어 | 크기 | 활성화 |
|--------|------|--------|
| Dropout | p=0.2 | - |
| Linear | flat_dim → 100 | ELU |
| Linear | 100 → 50 | ELU |
| Linear | 50 → 10 | ELU |
| Linear | 10 → 1 | 없음 (회귀) |

**입력 형식**: `(batch, 3, 66, 200)` (CHW, float32, 0~1 정규화)

## 이미지 전처리 파이프라인

### 사용 가능한 전처리 소스

| 소스 | 설명 | 리사이즈 필요 |
|------|------|--------------|
| `video` | 원본 이미지 | 예 |
| `cropped` | 상단 절반 크롭 | 예 |
| `invert` | CLAHE → 반전 → 가우시안 블러 | 아니오 |
| `otsu` | CLAHE → Otsu 임계값 | 아니오 |
| `adaptive` | CLAHE → 적응형 임계값 | 아니오 |
| `invert_clahe` | CLAHE → 반전 → CLAHE → 블러 | 아니오 |

### 전처리 단계 (10_Image_preprocessing.py)

1. **크롭**: 이미지 상단 절반만 사용
2. **그레이스케일 변환**: BGR → Grayscale
3. **CLAHE 적용**: 대비 제한적 히스토그램 평활화
4. **필터 적용**: 4가지 필터 중 선택
5. **리사이즈**: 200x66으로 축소
6. **정규화**: 0~1 범위로 정규화

## 데이터 증강 (9_make_model_aug.py)

6개 증강 기법이 학습 시 실시간으로 적용됩니다.

| 기법 | 확률 | 파라미터 | 각도 보정 |
|------|------|----------|-----------|
| 1. 수평 뒤집기 | 50% | - | `180 - angle` |
| 2. 랜덤 회전 | 50% | ±10° | 회전 각도만큼 더하기 |
| 3. 수평 이동 | 50% | ±15픽셀 | `(shift_x / width) * 20` |
| 4. 밝기/대비 | 50% | ±30% | 없음 |
| 5. 블러/노이즈 | 50% | 가우시안블러(5x5) 또는 노이즈(std=0.05) | 없음 |
| 6. 각도 지터 | 50% | ±2° | 미세 각도 변화 |

**적용 시점**: 학습 배치 생성 시에만 적용 (검증/테스트에는 미적용)

## 스티어링 각도 컨벤션

```
0도 (좌회전 최대)
│
│
270도 ──────── 90도 (직진) ──────── 180도 (우회전 최대)
(좌회전)        │              (우회전)
│
│
360도
```

- **90도**: 직진 (스티어링 휠 중앙)
- **90도 초과**: 우회전 (값이 클수록 더 많이 꺾음)
- **90도 미만**: 좌회전 (값이 작을수록 더 많이 꺾음)

**현재 데이터 한계**: 우회전만 학습됨 (90~179도), 좌회전 데이터 없음

## 개선 방향

### 데이터 수집
1. **좌회전 데이터 추가**: 90도 미만 각도 데이터 확보
2. **다양한 각도 데이터**: 각도별 최소50장 이상
3. **연속 프레임 수집**: temporal 연속성 확보
4. **다양한 환경**: 야간, 비, 안개 등 다양한 조건

### 모델 개선
1. **데이터 증강 확대**: 현재6개 기법 추가 적용
2. **하이퍼파라미터 튜닝**: 에포크, 학습률, 배치 크기 최적화
3. **전처리 파이프라인 개선**: 다양한 필터 조합 시도
4. **앙상블 모델**: 여러 모델 결합으로 성능 향상

### 평가
1. **교차 검증**: K-fold 교차 검증으로 일반화 성능 평가
2. **테스트 데이터 분리**: 학습/검증/테스트 3-way 분리
3. **실주행 테스트**: 실제 차량에서의 성능 검증
